# tfbox web catalog

Next.js 14 (App Router) static-export site that browses every Terraform module under `../aws/`. Reads `./public/catalog.json`, which is generated by `../scripts/build-catalog.ts`.

## Local development

```bash
# 1. generate catalog.json in web/public/
cd ../scripts && pnpm install && pnpm build:web

# 2. run dev server
cd ../web && pnpm install && pnpm dev
# http://localhost:3000
```

## Production build (static export)

```bash
pnpm build        # emits ./out/
```

`out/` is deployable to GitHub Pages, Vercel, S3+CloudFront, or any static host. For GitHub Pages under `<user>.github.io/tfbox`, `NEXT_PUBLIC_BASE_PATH=/tfbox` is set automatically in production.

## Regenerate the catalog

Anything in `../aws/**/*.tf` is picked up automatically on the next `pnpm catalog` (alias for `../scripts pnpm build:web`). No hand-editing required.

## Stack

- **Next.js 14** App Router, static export (`output: 'export'`).
- **Tailwind CSS** with custom token layer (see `app/globals.css`).
- **Radix UI** primitives for tabs + dialog.
- **Shiki** for HCL syntax highlighting (rendered at build time in RSC).
- **Fuse.js** for in-memory search across modules/inputs/outputs/resources.
- **react-flow + dagre** for the auto-laid-out module diagram.
- **cmdk** for the `⌘K` command palette.
- **next-themes** for dark (default) / light mode, respecting `prefers-color-scheme`.

## Routes

- `/`              — home catalog grid with filter + search.
- `/modules/[id]/` — per-module detail (tabs: Overview, Inputs, Outputs, Resources, Example, Diagram). Pre-rendered via `generateStaticParams`.

## Scripts

| script       | purpose                              |
|--------------|--------------------------------------|
| `pnpm dev`   | start local dev server               |
| `pnpm build` | static export → `./out`              |
| `pnpm catalog` | regenerate `public/catalog.json`   |
| `pnpm test`  | unit tests (vitest)                  |
| `pnpm test:e2e` | Playwright smoke test             |
